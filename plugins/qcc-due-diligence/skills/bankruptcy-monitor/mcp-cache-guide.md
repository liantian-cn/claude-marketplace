# MCP 查询缓存约定

> **适用范围**: 所有使用企查查 MCP 的 skill 均遵循此约定，以减少重复 MCP 请求。

## 缓存目录

每个公司一个目录：`./[公司全名]MCP查询结果/`

例如：`./小米科技有限责任公司MCP查询结果/`

## 缓存流程

执行任何 MCP 查询前，按以下步骤：

1. **检查缓存**: 检查 `./[公司名]MCP查询结果/[缓存文件名]` 是否存在
2. **命中**: 若文件存在且内容完整 → 直接读取，跳过 MCP 调用
3. **未命中**: 调用 MCP 工具 → 将返回结果以 Markdown 格式保存到对应缓存文件 → 继续使用

## 缓存文件格式

每个缓存文件包含：
```markdown
# [数据项名称]

**查询时间**: YYYY-MM-DD HH:MM
**数据来源**: [MCP 工具完整名]
**查询主体**: [公司全名] / [统一社会信用代码]

## 查询结果

[语义化的 MCP 返回内容]
```

## 缓存文件命名

详见本目录下的 `mcp-tools-map.md` 中的映射表。文件名格式为 `[序号]-[语义名称].md`。

## 缓存刷新策略

- 同一自然日内不重复请求同一数据项
- 如需强制刷新，删除对应缓存文件即可
